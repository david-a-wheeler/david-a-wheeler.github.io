MS Subverting Standards?

David A. Wheeler

September 17, 2004

Introduction

I'm glad when Microsoft works hard to improve their products. But lately, Microsoft seems to be working harder at manipulating standards to prevent competition. Isn't that illegal?

Let's look at the standards that Microsoft is trying to manipulate:

  1. Sender-ID, an anti-email-fraud technique. Microsoft patented their own technique, giving it a "free" licence that just happened to be essentially unusable by open source software (their primary competitor). http://news.zdnet.com/2100-3513_22-5369915.html (Microsoft's portion is called "Caller ID"). ??? Apache, Debian, IETF, Free Software Foundation analysis. IETF roundly reject this AOL rejected. http://www.groklaw.net/article.php?story=20040908180737547 http://www.groklaw.net/article.php?story=20040905212754195 http://www.imc.org/ietf-mxcomp/mail-archive/msg03514.html has detailed analysis of patent license vs. various freedom tests (including 'dissident' test, 'tentacles of evil' test, etc.)
  2. Copy protection. As noted elsewhere, http://news.com.com/Stalemate+on+digital+content%3F/2100-1025_3-5103601.html?tag=nl copy protection is wonderful for locking in customers. (RIAA move???)
  3. USB. http://www.pbs.org/cringely/pulpit/pulpit20040916.html September 16, 2004 Go to the Back of the Bus How Microsoft Plans to Use a New USB Standard to Cripple Linux By Robert X. Cringely Microsoft essentially controls the PC hardware standards, and exploiting this by developing a new USB standard that specifically excludes Linux and probably OS X devices as well. http://linux.slashdot.org/article.pl?sid=04/09/17/166235
