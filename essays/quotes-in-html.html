Curling Quotes in HTML and XML

by David A. Wheeler, 2025-10-08 (was 2002-12-10)

Summary

If you’re creating HTML or XML, use UTF-8 characters for curling quote characters. Here’s a table showing what I mean (you can copy and paste from this table if you would like):

History

I first wrote this page in 2002. At that time many character encodings were in wide use. A character encoding, if you do not know, is a system for converting human-readable text characters into numbers and back (so that characters can be stored and displayed).

In 2002 the UTF-8 character encoding had been created earlier, in 1992, but UTF-8 was used by less than 1% of all web (HTML) pages. HTML pages instead used a hodgepodge of different character encodings including ASCII, Western European encodings like Latin-1, (shift) JIS, Windows-1251, Windows-1252, and GB2312 among others. The default character encoding for HTML 4.0 was ISO-8859-1, aka Latin-1, though an HTML page could specify something different, and different they were. It was sometimes difficult to combine pages and fragments of pages because their character encodings were different.

Part of this came down to tooling. Many text editors and tools didn’t support UTF-8 natively in 2002. The first Linux using UTF-8 by default was Red Hat in 2002, which was a big deal at the time. Debian's default character encoding didn't switch to UTF-8 until 2007.

While this did cause many problems, the good news was that most character encodings used ASCII as a subset. So you could use ASCII characters to specify the other characters you wanted to display. This meant that the safest approach to display these characters was to use “decimal numeric character references” for curling single and double quote characters (these marks are called “smart quotes,” “curly quotes,” “curled quotes,” “curling quotes,” or “curved quotes”). In other words, for left and right double quotation marks, use &#8220; and &#8221; - and for left and right single quotation marks (and apostrophes), use &#8216; and &#8217;. This approach complies with all international standards, and works essentially everywhere. It still works today, as I expected. Thankfully, you no longer need to do it.

What’s changed?

In short, UTF-8 is now broadly supported and used. Over time, tools have added support for UTF-8. The current version of HTML defaults to UTF-8. Almost all HTML pages are in UTF-8. XML supports UTF-8, too. In 2002 I noted UTF-8, but I also noted how poorly UTF-8 was supported. I concluded in 2002, “For now, don’t do it.” Thankfully, things have changed for the better.
